Decimal, Binary, and Hexadecimal

Posted on 2016-03-19

Lately, I’ve been reading through some materials overviewing number systems, specifically in regards to computing. I’m going to jot down some of the reviewed ideas here.

The Decimal System

We’re all familiar with our Base 10 or decimal number system. It’s the most common number system globally and has been for quite some time. Evidence points to the first recorded usages of it in Egyptian Hieroglyphics dating back to ~3000 BC. Why 10? Probably because people tend to count using their fingers.

The idea behind the decimal system is simple. There are 10 symbols to work with:

Combined together, they can represent any quantity.

How do we represent quantity in a decimal system?

At a ridiculously simplified level, every number Base n number system works in a similar fashion. You count up using your available symbols 0.. 1.. 2.. (etc.) until you run out. When there are no more symbols left, a digit needs to be added. In order to get from 9 to 10, we flip the first (rightmost) digit back to our first symbol 0 and increase the newly added digit to the left 1 to leave us with 10.

In a Base 10 system, each digit represents a power of 10 (our base.) We count up in powers starting from the first (rightmost) digit.

Why do we use Binary in Computer Science?

Transistors are basic building blocks of microchips. They are tiny switches that enable or disable the flow of electricity by connecting or disconnecting two ends of wire. Learn more about them here. There are some lower-level details about this that we won’t delve into here but ultimately, these signal-triggered switches allow for very basic binary calculations. When combined together, these binary calculations can represent more complex chunks of data. If we wanted to represent 1,234 in binary, we would get 10011010010. The fundamental unit of memory inside a computer is called a bit (short for “binary digit.) These are the switches we just mentioned. Group together 8 bits in a row and you have a byte. A byte is considered the “smallest operable unit of information” in computer storage and processing. We’re all familiar with seeing bytes, kilobytes, megabytes, gigabytes, terabytes, etc. to measure computer hardware metics like hard drive capacity and RAM. Generally speaking, bits are not used as units of measurement unless we’re focusing on data transfer speed or networking. For instance, your internet speed is probably defined in terms of bits/second. (kbps = kilobits/second while kBps = kilobytes/second)

A byte can represent any decimal number from 0 to 255. Why is this? Well, if we apply our formula above to a byte we can prove this. Knowing that we have a maximum of 8 potential bits/digits and the max symbol in a binary system on each of these digits would be 1, we can convert 1111 1111 to decimal. Plug this into the formula above and you’ll see we get 255.

255 is found throughout computer science as a maximum value. The American Standard Code for Information Interchange (ASCII) is a standard which assigns letters, numbers, and characters to the 256 slots available in a byte (well, the extended version does.) Colors are represented as ranges of 0-255 of red green, and blue.

Hexadecimal

Hexadecimal notation or Base 16 is commonly found throughout computer science. The notation employs 16 symbols

Hexadecimal numbers can represent larger quantities in smaller strings than base 10 since we have a larger set of symbols to work through before having to move up a column.

Why hex and not decimal? The answer is convenience. Computers are built upon bytes but us humans have an easier time reading/remembering 1D rather than 0001 1101. You may be asking yourself, “Why can’t we just use decimal then?” Hex is easily convertible back to binary. Each hex digit directly maps to 4 binary bits. There is no direct mapping available between decimal numbers and groups of 4 or 8 bits.

Some Examples of Hexadecimal in Computing

Basic Encoding

Hexadecimal representations are often represented in C and UNIX along with many of their successors using the prefix 0x. Try opening your inspector window and typing 0x5A3. It will output 1443, a decimal conversion of hexadecimal 5A3.

Colors

HTML colors can be represented using hexadecimal systems. Colors in HTML use an additive color system, a mix of varying amounts of red, green, and blue. We can have a combination of these three primary colors that range of intensities from 0 (least intense) - 255 (most intense.) As we stated before, 255 is the maximum number a byte can store.

rgb(0, 0, 0) would be the darkest or least intense combination of the 3 primary colors (black) or #000000 (hex). rgb(255,255,255) is the lightest (white) or most intense. White can also be represented by #FFFFFF (hex).

Unicode

Unicode, which is a superset of ASCII, representings a euro sign would as 20AC. Try typing ’\u20ac’ into your inspector window. \u is a unicode escape character.

URL Encoding

Hex is also implemented in URL encoding. Try typing this in your browser navigation bar: http:google.com/these are spaces. The browser will convert the spaces to their UTF-8 value (32) and then to hexadecimal (20) and prefix them with the % that indicates a special character encoding. You should get http:google.com/these%20are%20spaces.
